## рҹҡҖ Quick Start

```bash
# 1. Clone & install
git clone https://github.com/V128S/TONMirror.git
cd TONMirror
pnpm install

# 2. Configure environment
cp .env.example .env.local
# вҶ’ fill in DATABASE_URL and other vars

# 3. Spin up the database
npx prisma migrate dev
npx prisma db seed      # loads 3 whale wallets + trade history

# 4. Launch
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) вҖ” no Telegram client required in dev mode.

## рҹ”җ Environment Variables

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string |
| `NEXT_PUBLIC_APP_URL` | Your public URL (TON Connect manifest) |
| `NEXT_PUBLIC_TONCONNECT_MANIFEST_URL` | Full path to `tonconnect-manifest.json` |
| `NEXT_PUBLIC_TELEGRAM_BOT_USERNAME` | BotFather bot username |
| `TON_WEBHOOK_SECRET` | Shared secret for live TON events |
| `TON_API_KEY` | Optional вҖ” live TON indexer key |
| `NEXT_PUBLIC_ENABLE_DEMO_MODE` | `"true"` вҖ” enables demo control panel |
| `NEXT_PUBLIC_ENABLE_LIVE_SOURCE` | `"true"` вҖ” activates live webhook source |

## рҹҸ— Architecture

```
app/          вҶ’ routes, layouts, screens (Next.js App Router)
components/   вҶ’ UI atoms: Button, Card, Badge, TabBar, BottomSheet
lib/          вҶ’ env, prisma singleton, format utils
modules/      вҶ’ pure TS domain logic (no framework imports)
  в”ңв”Җ strategy/        вҶ’ evaluation engine + filter rules
  в”ңв”Җ trade-ingestion/ вҶ’ MockLeaderTradeSource + TonWebhookTradeSource
  в”ңв”Җ omniston/        вҶ’ QuoteProvider adapter (STON.fi / Omniston)
  в””в”Җ execution/       вҶ’ PreparedExecution builder
server/       вҶ’ services, repositories, background jobs
prisma/       вҶ’ schema + idempotent seed
tests/        вҶ’ unit / integration / e2e
```

> All external integrations are hidden behind interfaces.  
> Swap the DEX, swap the indexer, swap the wallet вҖ” business logic doesn't care.
